## 🎯 ПРИНЦИПЫ ЯДРА

1.  **Не зависит от платформы** — нет импортов `telegram`, `max`, etc.
2.  **Возвращает данные, не сообщения** — платформа сама решает, как показать
3.  **`user_id` как `str`** — совместимость с разными платформами
4.  **Единая точка входа** — все обращения через `ChislyandiaEngine`
5.  **Баланс прежде всего** — эффекты имеют капы
6.  **Upkeep важен** — бонусы не работают без оплаты замка
7.  **Прямое обновление БД** — для `bank_balance`, `decoration_upgrades` минуя `save_user()`

# 🏰 ЧАСТЬ 4: ДЕКОРАЦИИ ЗАМКА (CASTLE DECORATIONS)

## 📌 НАЗНАЧЕНИЕ

**Декорации — это эндгейм-контент для долгосрочной мотивации.**

> *«Замок растёт вместе с игроком. Каждая декорация — шаг к величию!»*

**Проблема, которую решают декорации:**
- Игрок прокачал артефакты → нет новых целей
- Прогресс замедляется → мотивация падает
- Замок статичен → нет визуального прогресса

**Решение:**
- ✅ 8 декораций с прокачкой до 10 уровня = ~2.5 млн золота
- ✅ Бонус +2% за уровень, кап +10% на декорацию
- ✅ Максимальный бонус замка: +80% к очкам
- ✅ Зависимость от upkeep: бонусы не работают без оплаты
- ✅ ВладимИр управляет замком — нарративная связь

## 🔗 ЗАВИСИМОСТЬ ОТ UPKEEP

**Критически важно!** Бонусы декораций **НЕ РАБОТАЮТ** если upkeep замка не оплачен.

```python
# В castle_engine.py:
def _calculate_total_bonus(self, decoration_upgrades: Dict, bonuses_active: bool) -> float:
    if not bonuses_active:
        return 0.0  # ❌ Без upkeep = 0% бонусов!
    
    total_bonus = 0.0
    # ... расчёт бонусов
    return total_bonus
```

**Игровой поток:**
```
1. Игрок не оплатил upkeep замка
2. В /замок показывает: ❌ Upkeep НЕ оплачен!
3. Бонусы декораций: 0% (не применяются)
4. Игрок оплачивает upkeep: 💰 Оплатить 30 дней
5. В /замок показывает: ✅ Upkeep оплачен на 30 дн.
6. Бонусы декораций: +8% (применяются!)
```

### 1. База данных: колонка `decoration_upgrades`

-- Миграция: добавить колонку
ALTER TABLE users ADD COLUMN decoration_upgrades TEXT DEFAULT '{}'

### 2. Парсинг JSON в коде

```python
# Универсальная функция для парсинга
def _parse_decoration_upgrades(data):
    if data is None:
        return {}
    if isinstance(data, dict):
        return data
    if isinstance(data, str):
        try:
            return json.loads(data)
        except:
            return {}
    return {}

# Использование:
decoration_upgrades = _parse_decoration_upgrades(user.get("decoration_upgrades", {}))
level = decoration_upgrades.get("carrot_wall", 0)  # → 3
```

### 3. Прямое обновление БД в `upgrade_decoration()`

```python
def upgrade_decoration(self, user_id: str, decoration_id: str) -> Tuple[bool, str]:
    # ... проверка баланса ...
    
    # ✅ ПРЯМОЕ ОБНОВЛЕНИЕ БАЗЫ (минуя save_user!)
    conn = sqlite3.connect("data/progress.db")
    c = conn.cursor()
    
    # Получаем текущие уровни
    c.execute("SELECT decoration_upgrades FROM users WHERE user_id = ?", (str(user_id),))
    row = c.fetchone()
    current_upgrades = _parse_decoration_upgrades(row[0] if row else {})
    
    # Обновляем уровень
    new_level = current_level + 1
    current_upgrades[decoration_id] = new_level
    
    # Сохраняем в базу
    c.execute(
        "UPDATE users SET decoration_upgrades = ? WHERE user_id = ?",
        (json.dumps(current_upgrades, ensure_ascii=False), str(user_id))
    )
    conn.commit()
    conn.close()
    
    return (True, f"✅ Улучшено до уровня {new_level}!")
```

## ⚖️ БАЛАНС И ОГРАНИЧЕНИЯ

### Почему капы важны:

```
❌ Без капа:
• Декорация 10 уровня = +20% к очкам
• 8 декораций = +160% к очкам
• Экономика ломается, прогресс обесценивается

✅ С капом:
• Декорация 10 уровня = +10% к очкам (кап)
• 8 декораций = +80% к очкам (максимум)
• Прогресс чувствуется, но баланс сохраняется
```

### Формулы:

```python
# Стоимость улучшения
cost = base_price * (cost_multiplier ** current_level)
# Пример: 300 * (1.4 ** 5) = 300 * 5.38 = 1,613

# Бонус декорации
bonus = bonus_per_level + (bonus_per_level * (level - 1))
bonus = min(bonus, max_bonus)  # Кап!
# Пример: 0.02 + (0.02 * 9) = 0.20 → min(0.20, 0.10) = 0.10

# Общий бонус замка
total_bonus = sum(decoration_bonus for each decoration)
# Максимум: 8 × 10% = 80%
```
